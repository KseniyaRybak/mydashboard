# Дашборд по данным об абортах

Статический веб-дашборд для GitHub Pages. Он не требует backend: браузер загружает Parquet-файл, DuckDB-Wasm выполняет SQL-запросы, Plotly.js рисует графики.

## Структура проекта

```text
abortions-dashboard/
├── index.html
├── styles.css
├── app.js
├── README.md
└── data/
    └── abortions_long.parquet
```

## Как опубликовать на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите содержимое этой папки в корень репозитория.
3. Откройте `Settings` → `Pages`.
4. В `Build and deployment` выберите:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/ (root)`
5. Сохраните настройки и откройте опубликованную страницу.

## Что уже есть в дашборде

- фильтры по показателю, территории, типу организации, группе, возрасту и сроку беременности;
- KPI-блок с последним значением, годовым изменением, рейтингом территории и длиной временного ряда;
- график динамики;
- топ территорий за последний доступный год;
- кольцевая диаграмма возрастной структуры;
- таблица последних наблюдений;
- переключатель для показа значений `9 999 999`.

## Почему используется Parquet

В исходном архиве длинный CSV очень большой, а Parquet-файл компактный и быстро читается в браузере. Для GitHub Pages это гораздо практичнее.

## Локальный запуск

Открывать `index.html` напрямую через `file://` не стоит, потому что браузер может заблокировать загрузку Parquet-файла. Запустите простой локальный сервер:

```bash
python -m http.server 8000
```

Затем откройте страницу в браузере:

```text
http://localhost:8000
```
